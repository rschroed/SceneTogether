# UI system reference

For frontend work:

- use shared tokens
- use shared primitives
- avoid arbitrary raw values in feature code
- add screenshots for meaningful UI changes
- keep accents narrow and intentional

Preferred order:

1. tokens
2. primitives
3. composed features

Review for:

- visual hierarchy
- token usage
- pattern reuse
- mobile quality
- fidelity to the poster-inspired visual direction
