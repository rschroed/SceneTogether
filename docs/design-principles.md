# Design Principles

This document defines the visual bar for SceneTogether. It is meant to constrain design decisions, not merely inspire them.

## Visual references we are intentionally drawing from

The current inspiration set points toward:

- bold poster-like typography
- high-contrast, limited-palette compositions
- dark cinematic UI with vivid accent lighting
- screenprinted or punk-flyer energy
- graphic confidence rather than polished neutrality

We are not copying any single reference directly. We are extracting a design attitude.

## Product feeling

SceneTogether should feel:

- cinematic
- decisive
- electric
- legible
- social in energy, not in feed mechanics

It should not feel:

- like a generic productivity app
- like a movie database clone
- overly cute or novelty-driven
- chaotic enough to obscure decision-making

## Core visual thesis

The interface should feel like a movie-night decision board filtered through poster culture.

That means:

- one loud focal point
- strong graphic hierarchy
- limited but expressive color
- emotional energy without losing clarity

## Visual principles

### 1. Poster logic, product discipline

The references use poster composition: giant type, cropped imagery, dramatic contrast, and a small number of loud elements.

SceneTogether should borrow that composition logic while remaining usable as a product.

Use:

- oversized headings
- cropped poster art or strong image blocks
- sharp section hierarchy
- bold contrast between dominant and supporting content

Avoid:

- turning every screen into a decorative poster
- sacrificing scanability for aesthetic drama

### 2. Dark stage, neon signal

The mobile UI reference shows the right base model: dark surfaces with saturated accents.

Preferred visual setup:

- dark or near-black primary surfaces
- vivid accents used as signal, not wallpaper
- bright highlights for actions, confirmations, and ratings
- occasional atmospheric gradients at the shell level

Avoid:

- washed-out gray-on-gray interfaces
- rainbow accents with no hierarchy
- constant full-screen gradients behind content

### 3. Graphic confidence

The poster references share a willingness to be visually forceful.

That should show up in:

- typography with real presence
- deliberate scale jumps
- strong negative space
- high-contrast framing

Avoid:

- timid type hierarchies
- small safe headings
- over-reliance on subtle card grids

### 4. Controlled rawness

Some references have a hand-made, screenprinted, punk-flyer quality. That is useful for energy, but it should be translated carefully.

Allowed:

- textured backgrounds in contained doses
- slightly rough graphic moments
- editorial-style composition
- expressive accent shapes or stickers used sparingly

Avoid:

- fake distress everywhere
- messy layouts
- decorative grit that makes the interface harder to parse

### 5. Structure over feed behavior

UI should feel like a decision surface, not a stream of endless content.

Prefer:

- grouped actions
- visible group alignment
- clear next steps
- constrained interaction points

Avoid:

- infinite-feed composition
- chat-like dominance
- long, low-signal activity streams

### 6. Mobile-first, but visually authored

The strongest inspiration here is mobile-native. That matters.

The mobile layout should feel authored for the hand:

- bold header moments
- thumb-reachable primary actions
- strong vertical pacing
- cards and modules that feel curated, not stacked by default

Desktop should preserve the same design voice, not flatten it.

### 7. Strong defaults, limited freedom

Contributors should make fewer design decisions in feature work, not more.

That means:

- prefer approved tokens over new raw values
- prefer approved patterns over custom one-offs
- justify each new component pattern before introducing it

## Color direction

The reference set suggests a narrow, high-energy palette family:

- black / charcoal as the base
- electric magenta or hot pink as a primary accent family
- violet / indigo / cobalt as supporting accent families
- occasional acid yellow, signal red, or lime used as a secondary hit
- warm white instead of pure clinical white when possible

The palette should stay constrained. Too many accent colors will collapse the visual discipline.

## Typography direction

Typography should do a large share of the branding work.

Preferred characteristics:

- condensed or high-impact display faces for hero moments
- strong sans-serif system for UI text
- occasional editorial or hand-drawn attitude only if tightly controlled

Avoid:

- default system-only branding
- soft, neutral startup typography
- mixing too many expressive type voices

## Design review questions

Before approving UI work, ask:

1. What is the single dominant focal point?
2. Does the screen feel specific to movie-night coordination?
3. Is the design bold without becoming noisy?
4. Does color act as signal, not decoration?
5. Does the mobile layout still feel intentional?
6. Did this change introduce a new pattern unnecessarily?

## Anti-patterns

Reject UI that drifts into:

- default SaaS cards everywhere
- flat gray utility styling with no personality
- overusing chips, pills, and micro-badges
- weak typography hierarchy
- poster-inspired chaos without product discipline
- every section competing equally for attention

## Rule of thumb

If a screen could belong equally to a task manager, social app, or admin tool, it is not specific enough for SceneTogether.
